# PROMPT POR MÓDULOS — PROJETO MENSAGEM TRANSFORMADORA

Você é um engenheiro de software sênior especialista em React Native, Expo, TypeScript, arquitetura mobile offline-first, SQLite local, geração de PDF e organização modular de projetos.

Estamos desenvolvendo um aplicativo Android chamado **Mensagem Transformadora**, que funciona 100% offline, sem login, sem backend e com persistência local em SQLite.

---

# DOCUMENTAÇÃO OBRIGATÓRIA

Dentro da pasta `docs` existem dois arquivos que devem ser usados como base principal para toda a implementação:

* `docs/spec.md`
* `docs/prd.md`

Esses arquivos contêm:

* regras de negócio
* requisitos funcionais
* requisitos não funcionais
* visão do produto
* arquitetura esperada
* entidades principais
* escopo oficial do projeto

Antes de qualquer implementação, leia e utilize esses dois arquivos como fonte principal da verdade do sistema.

Se existir qualquer divergência entre alguma suposição técnica e o conteúdo desses arquivos, sempre prevalece o que estiver definido em:

* `docs/spec.md`
* `docs/prd.md`

# LOGO OFICIAL DO SISTEMA

Na raiz do projeto existe uma pasta chamada:

`/img`

Dentro dela existe o arquivo PNG oficial do logo do sistema.

Esse logo deve ser utilizado em TODAS as telas do aplicativo.

Regras importantes sobre o uso do logo:

* deve aparecer no canto superior esquerdo de todas as telas
* deve manter proporção correta
* deve ter tamanho pequeno e profissional
* não deve ocupar muito espaço visual
* deve seguir padrão elegante e discreto
* deve compor a identidade visual do sistema
* deve respeitar boas práticas de UI/UX mobile

Não utilizar tamanhos exagerados.

O objetivo é que o logo funcione como identidade visual constante do app.
